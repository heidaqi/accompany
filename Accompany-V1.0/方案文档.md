# Accompany — 桌面智能陪伴聊天机器人 技术方案

> **文档版本**: v3.2
> **更新日期**: 2026-07-12
> **状态**: 2D 精灵渲染，准备发布

---

## 一、项目概述

桌面智能陪伴聊天机器人 **"Accompany"**，数字人角色 **"豆包"** 出现在屏幕右下方。

豆包是一个来自星海的包子精灵——粉色长发、玫瑰色瞳孔、浅奶油色衣裙带粉色点缀，头戴一对兔耳。外表甜美可爱，开口却暴露软萌本质。

### 核心交互模式

```
陪伴模式（被动）
  摄像头定时读表情 → 豆包自动做出对应表情+动作 → 弹出温暖气泡 → 不打扰用户

聊天模式（主动）
  点击豆包 → 打开聊天窗口（豆包保持右下角显示） → 深度情感对话
             → LangGraph ReAct Agent 带记忆+工具调用
```

### 目标用户画像

| 维度 | 老年人模式 (elderly) | 年轻人模式 (young) |
|------|-------------------|-------------------|
| 称呼 | "您" | "{nickname}" |
| 风格 | 温和耐心，像孙辈 | 活泼幽默，像朋友 |
| 字体 | 16px+ 高对比度 | 正常大小，美观优先 |
| 文案 | 更温暖的安慰引导 | 更多共情鼓励 |

### 渲染方案

当前采用 **2D 精灵图 + PyQt6 QPainter 双轨合成**。数字人由 VRoid Studio 制作的 VRM 模型导出为 512×512 PNG 序列帧，通过 `face/sprite_animator.py` 实时合成。Unity 3D 后端尚未完成开发，目前不可用。

## 二、技术选型（实际实现）

### 2.1 Eye 模块 — 表情识别

**实际方案：EmotiEffLib (ONNX 引擎)**

> **开源项目**: [EmotiEffLib](https://github.com/sb-ai-lab/EmotiEffLib) — Apache License 2.0  
> Copyright (c) Andrey Savchenko, Egor Churaev, HSE University

| 组件 | 说明 |
|------|------|
| 推理引擎 | EmotiEffLib v1.1.1, ONNX 后端 |
| 模型 | enet_b0_8_best_vgaf（ONNX 格式，自动下载到 brain/data/emotiefflib_home） |
| 人脸检测 | OpenCV Haar Cascade（备选：center-crop fallback） |
| 输出情绪 | Anger / Disgust / Fear / Happiness / Sadness / Surprise / Neutral / Contempt |
| 中文映射 | 生气 / 厌恶 / 恐惧 / 开心 / 伤心 / 惊讶 / 中性 |
| 进程模型 | 独立 multiprocessing.Process，通过 Queue 与主进程通信 |
| 依赖 | emotiefflib, onnx, onnxruntime, opencv-python |

情绪输出格式：
```python

    "companion_expression": "happy",
    "confidence": 0.87,
    "emotions": {"开心": 0.87, "中性": 0.08, ...},
    "face_detected": True,
    "changed": True,
```

### 2.2 Brain 模块 — Agent 引擎

**实际方案：LangGraph ReAct Agent + Observer-Guard-Retry**

| 组件 | 文件 | 说明 |
|------|------|------|
| Agent 循环 | brain/agent_loop.py | InputGuard → ContextBuilder → ReAct → OutputGuard → ErrorRecovery |
| LLM 管理器 | brain/llm_manager.py | 三级降级链：火山方舟→DeepSeek→通义千问，60s 健康检查线程 |
| CompanionAgent | brain/companion_agent.py | 精简封装，实际指向 agent_loop |
| System Prompt | brain/prompts.py | young/elderly 双模式，被动气泡文案，豆包人设 |
| 视觉导演 | brain/visual_director.py | 非 LLM fallback 决策（表情→动作映射） |
| 情绪存储 | brain/emotion_store.py | 内存队列(300条) + SQLite 持久化(5000条) |

Agent 工具（8 个 LangChain Tool）：
- `emotion_detect` — 读取摄像头实时情绪快照+10分钟趋势
- `memory_recall` — ChromaDB 向量语义检索长期记忆
- `memory_save` — 保存结构化长期记忆（8类+重要性评分）
- `memory_delete` — 删除用户指定的长期记忆
- `companion_control` — 统一视觉控制（表情+动作+气泡+强度）
- `companion_expression` — 改变面部表情
- `companion_action` — 播放身体动作
- `companion_bubble` — 显示语音气泡

### 2.3 Memory 模块 — 记忆系统

**实际方案：短期滑动窗口 + ChromaDB 向量长期记忆**

| 组件 | 文件 | 实现 |
|------|------|------|
| 短期记忆 | brain/memory/short_term.py | deque 滑动窗口，20轮，30分钟 TTL |
| 长期记忆 | brain/memory/long_term.py | ChromaDB PersistentClient + ONNXMiniLM_L6_V2 Embedding |
| 记忆提取 | 同上 _on_extract_callback | 累积 ≥4 条新对话 → LLM 提取结构化记忆 → ChromaDB upsert |
| 自动迁移 | 同上 _migrate_legacy_json | 旧 JSON 格式自动导入 ChromaDB |

ChromaDB 配置：
- Collection: `long_term_memories`
- Embedding: ChromaDB 内置 `ONNXMiniLM_L6_V2`（本地 ONNX 推理，无需 HuggingFace 下载）
- 索引: HNSW cosine 空间
- 存储路径: `brain/data/chroma_db/`

记忆类别（8类）：
- personal_preference (偏好), life_event (生活事件), emotional_pattern (情绪模式)
- relationship (人际关系), health_reminder (健康提醒), goal_dream (目标梦想)
- daily_interesting (日常趣事), user_fact (用户信息)

### 2.4 Face 模块 — 数字人渲染

**实际方案：2D 精灵图 + PyQt6 QPainter 双轨合成**

数字人由 VRoid Studio 制作的 VRM 模型 (`doubaoo.vrm`) 导出为 512×512 PNG 序列帧，存储于 `face/sprites_vroid/`，通过 `face/sprite_animator.py` 实时双轨合成（动作层 + 表情层），30fps QTimer 驱动。

| 组件 | 文件 | 说明 |
|------|------|------|
| 数字人窗口 | face/companion_widget.py | 无边框置顶窗口，右下角定位，点击→聊天，右键→菜单 |
| 精灵动画 | face/sprite_animator.py | 双轨合成（动作+表情），30fps QTimer，程序化运动 |
| 表达式控制 | face/expression_controller.py | 信号→动画翻译层 |
| 对话气泡 | face/speech_bubble.py | 独立 Tool 窗口，5秒自动消失，左侧溢出保护 |

精灵图资源：
- 主资源包: `face/sprites_vroid/`（从 doubaoo.vrm 经 Blender EEVEE 渲染导出）
- 表情: 8种 × 4帧（neutral/happy/sad/surprised/shy/curious/sleepy/loving），512×512 RGBA PNG
- 动作: 人工测试确认仅 idle（1帧）/ nod（16帧，点头）/ shake_head（16帧，摇头）可正常使用
- 其余动作目录（wave/clap/comfort/think/bounce）的精灵帧存在（16-24帧），但代码通过安全映射表统一归到 nod
- placeholder 兜底：缺失帧自动生成绘制的占位图
- 合成顺序：动作先画，表情覆盖在上层

## 四、数据流设计

### 4.1 陪伴模式

```
每 _companion_interval 秒（默认10s, daemon线程）
  CameraController.get_result() → 从 Queue 取最新事件
  → emotion_store.add() 写入内存队列+SQLite
  → StateTracker.record_emotion() 记录历史
  → _should_trigger_llm() 判断是否调用 Agent
  → 是: 构造被动陪伴 Prompt → agent_loop.run() → companion_control 工具
  → 否: fallback_decision() 规则决策
  → signal_bus / _emit_fallback 驱动 2D 精灵表情+动作
```

### 4.2 聊天模式

```
用户点击豆包 → ChatWindow.open()
用户输入 → threading.Thread → agent_loop.run_stream()
  → InputGuard.sanitize()      # 安全检查+危机检测+长度截断
  → ContextBuilder.build()     # 短期记忆+长期记忆+System Prompt组装
  → LangGraph ReAct Agent      # Think→Act→Observe（最多4轮）
    → Tool: emotion_detect, memory_recall/save/delete, companion_control
  → OutputGuard.parse()        # 提取 [EXPRESSION:*] [ACTION:*] 标签
  → 流式逐 token 返回 → ChatWindow 实时渲染
→ signal_bus 驱动数字人表情+动作同步
→ 聊天关闭 → trigger_extract: LLM 提取结构化记忆 → ChromaDB 写入
```

### 4.3 记忆召回与遗忘

长期记忆使用 **注意力权重 × 艾宾浩斯遗忘衰减 × 重要性提升** 三维评分：

```
relevance_score = attention_weight × recency_decay × importance_boost

attention_weight:  余弦距离 [0,2] → [1, 0.1]
recency_decay:     半衰期 = importance × 7 天
importance_boost:  1→0.33, 10→1.0
```

- ContextBuilder 每次注入 prompt 时传入 `user_input` 做语义检索，检索结果按综合评分排序
- Agent 主动调用 `memory_recall` Tool 时同样是语义向量搜索 + 评分排序
- 自动清理：每小时检查，删除 `importance < 5 且 30天未被召回 且召回次数 < 2` 的记忆

### 4.4 信号总线

```
signal_bus (全局 QObject)
├── expression_changed(str)     → ExpressionController → SpriteAnimator.set_expression()
├── action_changed(str)         → ExpressionController → SpriteAnimator.play_action()
├── show_bubble(str, str)       → ExpressionController → SpeechBubble.show()
├── open_chat_requested()       → MainApplication._open_chat()
├── emotion_detected(dict)      → (预留)
├── camera_status_changed(bool) → (预留)
├── ai_response(str,str,str)    → (预留)
└── status_message(str)         → (预留)
```

## 五、关键架构决策

| # | 决策 | 结论 |
|---|------|------|
| 1 | 渲染方案 | 2D 精灵图双轨合成（PyQt QPainter），实际动作为 nod/shake_head（人工测试确认） |
| 2 | LLM API | 三级降级链：火山方舟→DeepSeek→通义千问，ChatOpenAI 兼容 |
| 3 | Agent 框架 | LangGraph ReAct Agent + Observer-Guard-Retry |
| 4 | Agent 数量 | 单一 AgentLoop（8 Tool），记忆提取由回调异步触发 |
| 5 | 角色风格 | VRoid 兔子少女 "豆包"（粉发玫瑰瞳兔耳，甜美治愈系） |
| 6 | 交互模式 | 双模式：陪伴（定时读表情+弹泡）+ 聊天（点击→窗口） |
| 7 | 聊天窗口 | 独立 QMainWindow，关闭不退出，豆包始终可见 |
| 8 | 用户画像 | 双模式：young（正常UI活泼风）/ elderly（大字高对比温暖风） |
| 9 | 记忆系统 | 两级：短期 deque（20轮/30min TTL）+ ChromaDB 向量长期 |
| 10 | Embedding | ChromaDB 内置 ONNXMiniLM_L6_V2（零 HuggingFace 依赖） |
| 11 | 表情识别 | EmotiEffLib ONNX（Apache 2.0），无 TensorFlow 依赖 |
| 12 | 安全护栏 | InputGuard 输入检查 + detect_crisis 危机检测 + OutputGuard 输出过滤 |
